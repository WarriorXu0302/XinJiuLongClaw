## 1. 环境搭建

### 1.1 基础设施

```bash
docker-compose up -d    # PostgreSQL(5433) + Redis(6379)
```

### 1.2 后端

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env    # 编辑配置
alembic upgrade head    # 应用数据库迁移
python -m app.scripts.seed   # 初始化种子数据
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

API 文档：http://localhost:8001/docs

### 1.3 前端

```bash
cd frontend
npm install
npm run dev    # http://localhost:5173（代理 /api → localhost:8001）
```

### 4.1 get_db — 带 RLS 上下文的数据库 session

```python

    # 用 erp_app 角色连接（受 RLS 约束）
```

### 4.3 CurrentUser — JWT 载荷

```python
CurrentUser = Annotated[dict[str, Any], Depends(get_current_user)]
# 包含：sub, username, roles, brand_ids, employee_id, is_admin, can_see_master
```

### 4.4 权限 helpers

```python
# permissions.py

is_privileged(user)            # admin/boss/sales_manager
is_salesman(user)              # 纯业务员
can_see_master_account(user)   # 看总资金池（仅 admin/boss）

```

## 8. 开发规范

### 8.1 后端

- Python 3.10+，async/await 全异步
- SQLAlchemy 2.0 声明式模型，`autoflush=False`
- **INSERT 后查聚合前必须 `await db.flush()`**
- Pydantic v2 做请求/响应验证
- 所有枚举集中在 `models/base.py`
- 审计日志用 `log_audit()` 服务
- 通知用 `notify()` / `notify_roles()` 服务
- 资金变动用 `record_fund_flow()` 记流水，**收款进 master 时必须显式传 brand_id**

### 8.2 前端

- TypeScript strict mode
- 函数式组件，Ant Design v6
- 数据获取用 React Query（useQuery / useMutation）
- 状态用 Zustand（authStore / brandStore）
- 金额统一 `Number(v).toLocaleString()`，**禁止 toFixed**
- 弹窗/详情 **禁止 Object.entries + JSON.stringify**，必须写专用 Descriptions
- 可展开行用 `expandIcon: () => null` + `expandRowByClick: true`（无加号）

### 8.3 数据库

- 迁移用 Alembic（生产）或直接 `docker exec psql`（开发）
- DDL 变更先在 Docker 里执行，再同步模型
- 新表如果含 brand_id 必须评估是否需要开 RLS
- `statement_cache_size=0` 防 asyncpg prepared statement 串 session

## 9. 新增功能开发流程

```
1. 明确需求 → 列出所有场景（三种结算模式分别怎么走？）
2. 定义字段 → 每个字段注释含义 + 各模式下的值
3. 写后端 → 一个业务动作一个原子接口
4. 改写入 → grep 全库所有读取方，逐一确认
5. 写前端 → 专用 Descriptions（禁止 JSON dump）
6. 端到端测试 → 从建单到完成走完整流程
7. 验证上下游 → 每个子任务检查向上+向下衔接
```
